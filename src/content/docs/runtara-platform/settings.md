# Settings

Settings contains account-level and organization-level controls exposed in the web application.

## API Keys

Open Settings, then API Keys, to manage keys used by external systems or automation clients.

API keys allow external clients to call Runtara:

- Create a key for each external integration or automation client.
- Store the key in the system that will call Runtara.
- Replace a key when ownership or access requirements change.
- Delete unused keys when an integration is retired.

## Billing and Account Controls

Depending on deployment configuration, the sidebar may expose billing or account-management controls. These controls are separate from workflow configuration and should be handled by an administrator.

## Theme and Layout

The bottom-left controls include theme and sidebar layout actions. These affect the current UI experience and do not change workflow behavior.
