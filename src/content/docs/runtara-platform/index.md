# Runtara Platform User Guide

This guide explains the main Runtara product areas from the perspective of an operator, builder, or analyst using the web application.

The screenshots in this guide were captured at desktop width and 2x scale so they remain readable on retina displays.

## Product Areas

| Area | Use it for | Guide |
| --- | --- | --- |
| Getting started | First tour of the app, navigation, and demo data | [Getting started](/docs/runtara-platform/getting-started) |
| Workflows | Build, validate, run, and inspect workflow automations | [Workflows](/docs/runtara-platform/workflows) |
| Triggers | Start workflows from HTTP calls, schedules, and channel events | [Triggers](/docs/runtara-platform/triggers) |
| Connections | Configure provider credentials and shared integrations | [Connections](/docs/runtara-platform/connections) |
| Database | Manage object types and records used by workflows and reports | [Database](/docs/runtara-platform/database) |
| Reports | View, edit, explore, print, and publish operational reports | [Reports](/docs/runtara-platform/reports) |
| Files | Browse files created or used by workflow runs | [Files](/docs/runtara-platform/files) |
| Invocation history | Review executions across workflows and triggers | [Invocation history](/docs/runtara-platform/invocation-history) |
| Analytics | Monitor usage, system health, and rate limits | [Analytics](/docs/runtara-platform/analytics) |
| Settings | Manage API keys and account-level controls | [Settings](/docs/runtara-platform/settings) |
| Troubleshooting | Diagnose common workflow, trigger, report, and connection issues | [Troubleshooting](/docs/runtara-platform/troubleshooting) |

## Main Navigation

Runtara uses a persistent sidebar. Workflows, Database, Reports, Files, Triggers, Connections, and Analytics are top-level product areas. Workflow folders appear below the Workflows entry so teams can keep automations organized by function.

![Runtara workflows list](/docs/runtara-platform/images/workflows-list.png)

## Suggested Learning Path

1. Start with [Getting started](/docs/runtara-platform/getting-started) to learn the layout and demo organization.
2. Read [Workflows](/docs/runtara-platform/workflows) and [Triggers](/docs/runtara-platform/triggers) together. Workflows define what happens; triggers define when they start.
3. Use [Connections](/docs/runtara-platform/connections), [Database](/docs/runtara-platform/database), and [Files](/docs/runtara-platform/files) as supporting references while building.
4. Use [Reports](/docs/runtara-platform/reports), [Invocation history](/docs/runtara-platform/invocation-history), and [Analytics](/docs/runtara-platform/analytics) to inspect results and operations.
5. Keep [Troubleshooting](/docs/runtara-platform/troubleshooting) nearby when something does not render, run, or connect as expected.
